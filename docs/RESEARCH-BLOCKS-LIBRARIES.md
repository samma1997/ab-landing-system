## CONVERSION DESIGN PATTERNS (Research 2025-2026)

### Pattern ad Alta Conversione Documentati

1. **CLEAR Framework**: Clarity, Layout, Engagement, Action, Resonance
   - Above-the-fold: 1 headline + 1 subhead + 1 CTA
   - F-pattern o Z-pattern per scanning naturale

2. **Single CTA Focus**: +62% conversioni con un singolo CTA focalizzato

3. **Form Length Reduction**: -120% lift in conversioni riducendo campi form

4. **Headline Optimization**: 27-104% improvement con headline testing

5. **Strategic Proof Placement**: Testimonianze PRIMA del CTA, non dopo
   - Matching testimonianze con obiezioni specifiche del visitatore

6. **Mobile-First**: Headline + CTA visibili immediatamente su mobile

7. **Multiple Landing Pages**: 31-40 landing page = 7x piu lead vs 1-5 pagine
   - Landing page diverse per fonte di traffico

8. **Visual Hierarchy**: White space + CTA button contrastante + layout pulito

## RIEPILOGO — Cosa Usare per ab-landing-system

### Stack Consigliato per i Blocchi

| Categoria | Libreria Primaria | Alternativa |
|-----------|------------------|-------------|
| **Componenti Animati** | Magic UI + Aceternity UI | React Bits, Motion Primitives |
| **Blocchi Sezioni** | Launch UI + Page UI | Shadcnblocks, HyperUI |
| **Base UI** | Shadcn/ui | — |
| **Animazioni** | Motion (Framer Motion) | GSAP per effetti complessi |
| **Styling** | Tailwind CSS v4 | — |
| **Framework** | Next.js 15+ | — |

### Blocchi Necessari (Checklist)

- [ ] Hero Section (5+ varianti: centered, split, video, animated, minimal)
- [ ] Trust Bar / Logo Cloud
- [ ] Features Grid (icon grid, bento grid, alternating)
- [ ] Social Proof / Testimonials (carousel, grid, single highlight)
- [ ] Pricing Tables (2-3 colonne, toggle monthly/annual)
- [ ] FAQ Accordion
- [ ] CTA Section (inline, full-width, floating)
- [ ] Footer (multi-column, minimal, with newsletter)
- [ ] Stats / Numbers Section
- [ ] How It Works / Steps
- [ ] Comparison Table
- [ ] Video Showcase
- [ ] Newsletter / Lead Capture Form

### Approccio Architetturale

1. **JSON-driven content**: Contenuti delle landing page definiti in JSON (pattern da Landy)
2. **Variant system**: Ogni blocco ha multiple varianti selezionabili (pattern da Launch UI)
3. **Copy-paste components**: Componenti che si copiano nel progetto, non dipendenze npm (pattern da Shadcn)
4. **Animation layers**: Animazioni come layer separato applicabile a qualsiasi blocco (pattern da Motion Primitives)
5. **A/B testing ready**: Struttura che permette di swappare varianti per testing
